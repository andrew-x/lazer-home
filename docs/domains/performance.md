# Domain: Performance management

**Status: proposed.** Periodic review and growth tracking for People, informed by their delivery work.

## Purpose

Run review cycles, set and track goals, and ground assessment in real signals (project work, utilization) rather than memory.

## Key entities

- **ReviewCycle** — a period in which reviews happen (e.g. quarterly, annual).
- **PerformanceReview** — a Person's assessment within a cycle; may pull in their project work and utilization.
- **Goal** — an objective for a Person, tracked over time.

## Key flows

- **Review cycle** — open cycle → collect input (self, manager, peers?) → assessment → close.
- **Goal setting & tracking** — set goals, check progress across cycles.
- **Evidence pull** — surface a Person's allocations, utilization, and project contributions as review context.

## Connects to

- **Staff profiles** — the Person being reviewed; outcomes may update role/seniority.
- **Timesheets / Allocations** — utilization and delivery are review inputs.

## Open questions

- Review types: self / manager / 360?
- How tightly do utilization metrics factor into ratings (and who can see them)?
- Cycle cadence and rating scale.
